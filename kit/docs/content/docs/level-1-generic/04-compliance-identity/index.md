# Regional Compliance

**The DALP treats regulation as code by shipping jurisdiction templates, eligibility rule packs, and audit evidence bundles so cross-border programmes stay compliant from day one.**

## How to Use This Section

- [European Union — MiCA Playbook](./regional-guides/europe-mica) maps €8 million issuance caps, whitepaper workflows, and reserve requirements into automated rule sets.
- [United States — SEC Exemptions](./regional-guides/us-sec) shows how Regulation D/S/CF/A+ and Rule 144 holding periods are enforced before every transfer.
- [Singapore — MAS Compliance Roadmap](./regional-guides/singapore-mas) documents PSA/SFA classifications, Travel Rule enforcement, and sandbox options.
- [GCC — Regional Regulatory Matrix](./regional-guides/gcc-regulations) covers ADGM, DFSA, VARA, CMA, and CBB licensing models with deployment guardrails.
- [Identity & ERC-3643 — OnchainID](./identity-onchainid) explains how verified claims, role assignments, and revocation events stay synchronised with compliance logic.

## Implementation Themes

- **Template-first configuration:** Pre-built YAML rule packs translate MiCA, SEC, MAS, and GCC policies into eligibility checks, thresholds, and disclosure gates without custom code (Appendix D).
- **Evidence on tap:** Each regional guide pairs operational workflows with audit artefacts so regulators see immutable proof of policy enforcement (Part IV Ch 20).
- **Identity anchored in ERC‑3643:** OnchainID claims and identity registries give institutions a consistent way to assert investor status, trusted issuer roles, and revocation events across jurisdictions (Part II Ch 8).

Select the jurisdiction relevant to your programme, apply the provided templates, and the platform enforces the right guardrails before any asset changes hands.
